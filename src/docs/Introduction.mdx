# Welcome to Help Panel Storybook

Welcome to the **Red Hat Console Help Panel UI** component library! This Storybook showcases all the components, design patterns, and documentation for the contextual help and learning system.

## 🎯 What is the Help Panel?

The Help Panel is a comprehensive assistance system that provides users with contextual help, learning resources, and support options directly within the Red Hat Console interface. It includes:

- **Learn Tab** - Access to learning resources, quick starts, and documentation
- **Search Functionality** - Find relevant help content across all Red Hat services
- **API Documentation** - Interactive API documentation and examples
- **Support Integration** - Direct access to support resources and ticket creation
- **Feedback System** - Collect user feedback and improve the experience

## 🏗️ Architecture Overview

The Help Panel is built with modern React patterns and integrates seamlessly with the Red Hat Console platform:

### Key Technologies
- **React** with functional components and hooks
- **PatternFly 6** for design system and components
- **React Intl** for internationalization
- **MSW (Mock Service Worker)** for API mocking in Storybook
- **TypeScript** for type safety
- **Storybook v9** for component development and documentation

### Integration Points
- **Red Hat Console Chrome** - Platform integration and navigation
- **Learning Resources API** - Quick starts, tutorials, and documentation
- **User Preferences** - Bookmark management and personalization
- **Analytics** - User behavior tracking and optimization
- **Feature Flags (Unleash)** - Conditional feature rendering

## 🎨 Design Principles

### Contextual Relevance
- Content is filtered based on current application context
- Learning resources adapt to user's current workflow
- Help suggestions are relevant to the active page or feature

### Progressive Disclosure
- Information is presented in digestible layers
- Quick access to essential help with details on demand
- Advanced options available for power users

### Accessibility First
- Full keyboard navigation support
- Screen reader compatibility with proper ARIA labels
- High contrast mode support
- Responsive design for all screen sizes
