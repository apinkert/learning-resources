# Help Panel Storybook Guide

This guide explains how to create effective Storybook stories for Help Panel components and best practices for documenting the contextual help system.

## 🎨 What Makes This Storybook Special

### Help Panel Focus
- **Contextual Stories** - Stories that demonstrate components in realistic application contexts
- **Interactive Learning Flows** - Complete user journeys from discovery to completion
- **Multi-State Testing** - Components tested across different user permissions and contexts
- **Responsive Testing** - Help panel behavior across different screen sizes

### Enhanced Development Experience
- **MSW Integration** - Realistic API responses for learning resources
- **Context Simulation** - Test components in different bundle contexts (Insights, Automation, etc.)
- **Permission Testing** - Validate component behavior for different user roles
- **Internationalization Testing** - All text content properly localized

## 🎯 Story Development Best Practices

### **Focus on User Journeys**

Help Panel stories should demonstrate complete user workflows:

```typescript
// ✅ Good: User journey focused
export const LearnTabDiscoveryFlow: Story = {
  name: 'Discovery to Learning Flow',
  parameters: {
    docs: {
      description: {
        story: `
          Demonstrates the complete user journey from discovering learning resources
          to interacting with content in the Learn tab.
        `
      }
    }
  },
  play: async ({ canvasElement }) => {
    const canvas = within(canvasElement);

    // User opens help panel
    await userEvent.click(canvas.getByLabelText('Open help panel'));

    // User explores content filters
    await userEvent.click(canvas.getByLabelText('Content type'));
    await userEvent.click(canvas.getByText('Quick starts'));

    // User bookmarks a resource
    await userEvent.click(canvas.getByLabelText(/bookmark.*getting started/i));

    // User opens the resource
    await userEvent.click(canvas.getByText('Getting Started with Red Hat Insights'));
  }
};
```

## 🧪 Interactive Testing Strategies

### **MSW API Mocking**

```typescript
// Comprehensive API mocking for realistic testing
const handlers = [
  // Learning resources endpoint
  http.get('/api/quickstarts/v1/quickstarts', () => {
    return HttpResponse.json(mockLearningResources);
  }),

  // User progress tracking
  http.get('/api/quickstarts/v1/progress/:user', ({ params }) => {
    return HttpResponse.json({
      [params.user]: {
        'getting-started-insights': {
          status: 'In Progress',
          taskNumber: 2
        }
      }
    });
  }),

  // Bookmark management
  http.post('/api/quickstarts/v1/favorites', async ({ request }) => {
    const body = await request.json();
    return HttpResponse.json({
      success: true,
      quickstartName: body.quickstartName,
      favorite: body.favorite
    });
  }),

  // Error scenarios
  http.get('/api/quickstarts/v1/quickstarts', () => {
    return new HttpResponse(null, { status: 500 });
  }, { once: true })
];
```

### **State Management Testing**

```typescript
// Test bookmark state changes
export const BookmarkInteraction: Story = {
  play: async ({ canvasElement }) => {
    const canvas = within(canvasElement);

    // Initial state: resource not bookmarked
    const bookmarkBtn = canvas.getByLabelText(/bookmark.*getting started/i);
    expect(bookmarkBtn).toHaveAttribute('aria-pressed', 'false');

    // User bookmarks the resource
    await userEvent.click(bookmarkBtn);

    // Wait for API call and state update
    await waitFor(() => {
      expect(bookmarkBtn).toHaveAttribute('aria-pressed', 'true');
    });

    // Test bookmark filter
    await userEvent.click(canvas.getByLabelText('Show bookmarked only'));

    // Verify only bookmarked items are shown
    await waitFor(() => {
      expect(canvas.getByText('Getting Started with Red Hat Insights')).toBeInTheDocument();
      expect(canvas.queryByText('Advanced Configuration Guide')).not.toBeInTheDocument();
    });
  }
};
```

### **Screen Reader Testing**

```typescript
export const ScreenReaderExperience: Story = {
  parameters: {
    docs: {
      description: {
        story: `
          Tests the help panel experience for screen reader users:
          - Proper heading structure
          - ARIA labels and descriptions
          - Live regions for dynamic content updates
          - Semantic markup for lists and navigation
        `
      }
    }
  },
  play: async ({ canvasElement }) => {
    const canvas = within(canvasElement);

    // Verify semantic structure
    expect(canvas.getByRole('main')).toBeInTheDocument();
    expect(canvas.getByRole('navigation')).toBeInTheDocument();
    expect(canvas.getByRole('list', { name: /learning resources/i })).toBeInTheDocument();

    // Test live region updates
    await userEvent.click(canvas.getByLabelText('Content type'));
    await userEvent.click(canvas.getByText('Quick starts'));

    const liveRegion = canvas.getByRole('status');
    await waitFor(() => {
      expect(liveRegion).toHaveTextContent(/learning resources.*updated/i);
    });
  }
};
```

## 📋 Story Checklist

Before submitting a Help Panel story, ensure:

### ✅ **Functionality**
- [ ] Component renders without errors
- [ ] All interactive elements work as expected
- [ ] API integration is properly mocked
- [ ] Error states are handled gracefully

### ✅ **Accessibility**
- [ ] Keyboard navigation works correctly
- [ ] Screen reader experience is optimized
- [ ] Color contrast meets WCAG standards
- [ ] Focus management is appropriate

### ✅ **User Experience**
- [ ] Realistic user scenarios are covered
- [ ] Loading and error states are included
- [ ] Responsive behavior is tested
- [ ] Performance is acceptable with large datasets

### ✅ **Documentation**
- [ ] Component purpose is clearly explained
- [ ] Usage examples are provided
- [ ] Props are fully documented
- [ ] Edge cases are noted
