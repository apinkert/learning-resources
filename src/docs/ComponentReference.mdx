# Component Reference

This document provides detailed reference information for Help Panel components.

## 📦 Component Index

### Core Components

- **HelpPanelContent** - Main container for the Help Panel

## HelpPanelContent

**Location:** `src/components/HelpPanel/HelpPanelContent.tsx`

Main component that provides the Help Panel header, status page link, and tab container.

### Props

```typescript
interface HelpPanelContentProps {
  toggleDrawer: () => void;
  newTab?: HelpPanelTabContent;
}
```

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `toggleDrawer` | `() => void` | Yes | Function to close the Help Panel drawer |
| `newTab` | `HelpPanelTabContent` | No | Content to open in a new tab |

### Usage

```typescript
<HelpPanelContent toggleDrawer={handleCloseDrawer} />
```

### Features

- Displays "Help" title with status page link
- Status page link is always visible in the header
- Provides drawer close button
- Contains HelpPanelCustomTabs for tab navigation

## HelpPanelCustomTabs

**Location:** `src/components/HelpPanel/HelpPanelCustomTabs.tsx`

Manages tab lifecycle, state, and navigation.

### Ref Interface

```typescript
export type HelpPanelCustomTabsRef = {
  openTabWithContent: (content: HelpPanelTabContent) => void;
};
```

### Methods (via ref)

#### `openTabWithContent(content: HelpPanelTabContent)`

Opens a tab with custom content. If a tab with the same ID exists, it updates that tab; otherwise, creates a new one.

**Parameters:**
```typescript
type HelpPanelTabContent = {
  id?: string;        // Optional unique ID
  title: string;      // Tab title
  tabType: TabType;   // Content type
  content: ReactNode; // Custom content to render
};
```

**Example:**
```typescript
const tabsRef = useRef<HelpPanelCustomTabsRef>(null);

const openCustomTab = () => {
  tabsRef.current?.openTabWithContent({
    id: 'my-custom-tab',
    title: 'Custom Help',
    tabType: TabType.learn,
    content: <CustomHelpContent />,
  });
};

return <HelpPanelCustomTabs ref={tabsRef} />;
```

### Internal State

```typescript
// Tab definition structure
type TabDefinition = {
  id: string;
  title: ReactNode;
  tabTitle?: string;
  closeable?: boolean;
  tabType: TabType;
  isNewTab?: boolean;
  customContent?: ReactNode;
  quickstartId?: string;
};
```

### Tab Lifecycle

1. **Creation:**
   - Base tabs created on mount
   - Dynamic tabs created via "Add tab" button or `openTabWithContent`
   - Quick Start tabs created via `openQuickstartInHelpPanelStore`

2. **Updates:**
   - Title updates when user performs actions (debounced 100ms)
   - Tab type updates when user switches sub-tabs
   - Content updates for custom content tabs

3. **Deletion:**
   - User closes tab via close button
   - Quick Starts in progress require confirmation modal
   - Cannot close base tabs (VA, Find Help)

### Feature Flags

- `platform.chrome.help-panel_chatbot` - Virtual Assistant tab visibility

## HelpPanelTabContainer

**Location:** `src/components/HelpPanel/HelpPanelTabs/HelpPanelTabContainer.tsx`

Routes active tab type to the appropriate panel component.

### Props

```typescript
interface HelpPanelTabContainerProps {
  activeTabType: TabType;
  setNewActionTitle: (title: string) => void;
  customContent?: ReactNode;
}
```

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `activeTabType` | `TabType` | Yes | Current active tab type |
| `setNewActionTitle` | `(title: string) => void` | Yes | Callback to update tab title |
| `customContent` | `ReactNode` | No | Custom content to render instead of default panel |

### Implementation

```typescript
const HelpPanelTabContainer = ({
  activeTabType,
  setNewActionTitle,
  customContent,
}) => {
  // If custom content provided, render it
  if (customContent) {
    return <>{customContent}</>;
  }

  // Otherwise route to appropriate panel component
  const PanelComponent = helpPanelTabsMapper[activeTabType];
  return <PanelComponent setNewActionTitle={setNewActionTitle} />;
};
```

## HelpPanelLink

**Location:** `src/components/HelpPanel/HelpPanelLink.tsx`

Component that allows external components to open custom tabs in the Help Panel.

### Props

```typescript
interface HelpPanelLinkProps {
  content: HelpPanelTabContent;
  children: ReactNode;
  onClick?: () => void;
}
```

### Usage

```typescript
<HelpPanelLink
  content={{
    id: 'api-endpoint-details',
    title: 'POST /api/users',
    tabType: TabType.api,
    content: <APIEndpointDetails endpoint="/api/users" />,
  }}
>
  View API details
</HelpPanelLink>
```

### Behavior

1. Renders children as a clickable element
2. On click, opens Help Panel if closed
3. Calls `openTabWithContent` on HelpPanelCustomTabs ref
4. Optionally calls custom `onClick` handler

## Stores

### openQuickstartInHelpPanelStore

**Location:** `src/store/openQuickstartInHelpPanelStore.ts`

Global store for coordinating Quick Start opening from external components.

**State:**
```typescript
type OpenQuickstartInHelpPanelState = {
  pendingOpen?: {
    quickstartId: string;
    displayName: string;
  };
};
```

**Methods:**
- `updateState('OPEN', data)` - Request Quick Start open
- `updateState('CONSUMED_OPEN')` - Mark request as consumed

**Usage:**
```typescript
// From catalog or other component
const store = getOpenQuickstartInHelpPanelStore();
store.updateState('OPEN', {
  quickstartId: 'my-quickstart',
  displayName: 'My Quick Start',
});

// Help Panel listens and responds
useEffect(() => {
  const { pendingOpen } = openQuickstartState;
  if (!pendingOpen) return;
  // Handle open...
  store.updateState('CONSUMED_OPEN');
}, [openQuickstartState.pendingOpen]);
```

## Performance Considerations

### Lazy Loading

- Quick Starts loaded asynchronously after user authentication
- Remote modules (VA) loaded on demand
- Tab content only rendered when active

### Debouncing

```typescript
// Tab title updates debounced to 100ms
const setNewActionTitleDebounced = useCallback(
  debounce((title: string) => {
    // Update logic
  }, 100),
  [dependencies]
);
```

### Memoization

```typescript
// Base tabs memoized based on feature flag
const baseTabs = useMemo(() => createBaseTabs(vaFlag), [vaFlag]);

// Filtered sub-tabs memoized
const filteredSubTabs = useMemo(() => {
  return subTabs.filter(/* ... */);
}, [flags, subTabs]);
```
